## Product 정보 — Job Search Co-pilot

> **"don't make hiring teams connect the dots for you"**

| 항목 | 내용 |
|------|------|
| Product 이름 | Job Search Co-pilot |
| 플랫폼 | Claude Cowork plugin |
| GitHub | https://github.com/nathanscho/jobsearch-copilot |
| LinkedIn 글 | https://www.linkedin.com/posts/nathanscho_interviewing-communication-productmanagement-share-7478447354132525056-OR-g/ |
| 상태 | Free, Open source (MIT), 초기 테스트 중 (소수 지인과 테스트) |
| 사용 조건 | Claude 계정 (Cowork access) + 이력서 |

### 해결하려는 문제

기존 취업 준비 도구들은 "더 빠르게, 더 많이" (지원 수, 네트워킹 수, 키워드 최적화)에 초점. Nate는 반대 방향 — **어떤 역할이 나에게 맞는지 파악하고, 그 이유를 내가 먼저 정리해서 면접관에게 제시하는 것**이 더 강력하다는 원칙으로 만들었다.

### 핵심 기능

- **Positioning** — 역할이 실제로 어떤 문제를 해결하려는지 파악하고, 나만의 포지셔닝 thesis 구성
- **Story bank** — 경험을 이야기 형식으로 정리
- **Interview prep briefs** — 인터뷰별 맞춤 준비 자료 생성
- **Interview debriefs** — 실제 인터뷰 후 회고 정리
- **자동화 옵션** — 이메일 계정 연동 시 job discovery + pipeline status 자동 추적

### 차별화 포인트

- 지원 수가 아닌 **fit 식별**과 **positioning 명확화** 중심
- 사용할수록 나의 실제 컨텍스트가 쌓여서 점점 정확해지는 구조
- Senior PM 및 유사 직군 중심으로 설계됐지만 다른 직군으로 확장 가능

## Builders Lounge 연결 포인트

Nate는 직접 만든 Product를 사람들에게 테스트시키길 원하고 있다. 이것이 Builders Lounge가 하려는 일과 정확히 일치한다.

| BL 구상 | Nate의 상황 |
|---------|------------|
| Builder가 Product를 소개하고 Feedback을 받는 공간 | 초기 테스트 단계, 실제 피드백 필요 |
| 멤버들이 서로의 Product를 써보는 커뮤니티 | 취업 준비 중인 멤버들이 있다면 바로 사용 가능 |
| Catch Up AI에서 Builder 인터뷰 / Product 소개 | 취업 준비 → AI 활용 각도로 흥미로운 인터뷰 소재 |
| Bila AI가 니즈 있는 멤버와 Product를 매칭 | Bila가 BL 내 구직 중인 멤버에게 추천 가능 |

**추가 시너지**: Job Search Co-pilot은 Claude Cowork plugin이라서, Claude Code와 AI 활용 생태계를 잘 이해하는 Builders Lounge 멤버들과 기술적 대화도 자연스럽게 이어질 수 있다.
