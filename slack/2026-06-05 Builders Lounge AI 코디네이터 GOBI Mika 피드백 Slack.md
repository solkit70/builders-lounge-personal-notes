## 핵심 인사이트 정리

### Phase 1 (QnA/reactive) — 현재 구현 가능

- GOBI 플랫폼에서 지금 바로 시작 가능 (기존 기능으로 충분)
- 창발빌라 목적에 맞는 프롬프트와 데이터 정제가 관건
- 디자인/구현 방향은 Mika와 함께 논의 예정

### Phase 3 (커뮤니티 코디네이터) — 기술 구현 방향

**미팅 스케줄링 & 로그 자동화 아키텍처:**
- 어드민의 Google Workspace 연결 → 에이전트에 미팅 스케줄 권한 부여
- Google Meet 종료 → Gemini 미팅 노트 생성 → **웹훅 트리거** → 에이전트 활성화
  - 미팅 노트 기반 자동 포스팅 가능
  - 다음 미팅 자동 예약 가능
  - Gemini가 녹취 역할 담당 → 별도 녹취 에이전트 불필요

**권한 관리 제약:**
- 모든 그룹 멤버에게 Google Workspace 권한 부여 불가
- **Space Agent ↔ 어드민 전용 대화 공간** 별도 필요

### Phase 2 (유저 연결) — Cron 트리거 방식

- 시간 기반 Cron 트리거 추가로 구현 가능
- 기능 예시:
  - 토픽별 최근 글 리서치 → 관련 유저 자동 연결
  - 커뮤니티 활동 침체 시 업데이트 포스트 자동 게시

### Mika의 구조적 분류

기능을 두 축으로 정리:

| 축 | 설명 | 예시 |
|----|------|------|
| **툴/권한** | 에이전트에게 부여할 외부 시스템 접근 권한 | Google Workspace 미팅 스케줄 |
| **트리거** | 에이전트를 활성화하는 이벤트 | Gemini 미팅 노트 생성 / Cron / 외부 트리거 |
